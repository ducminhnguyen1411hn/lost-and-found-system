# Feature Records

One markdown file per finished feature (e.g. `found-item-create.md`, `claim.md`,
`matching-subscribe.md`). Each captures **lessons learned** + a **reusable recipe** so the next
similar screen is faster to build.

- Generate one with the **`/feature-record`** skill (it fills the template).
- Writing the record is part of the **Definition of Done** — do it when the feature is fresh, not at
  end of term.
- Link the record in the PR description.

See the template and rationale in `docs/FEATURE_PLAYBOOK.md` (once you add it) and `CLAUDE.md`.
